## 项目概述

校园球馆预约系统是一个面向高校师生的体育场馆在线预约平台，支持羽毛球、篮球、乒乓球、网球、排球等多种运动类型的场地预约。系统采用前后端分离架构，包含管理后台和微信小程序两个前端应用。

### 主要功能

- **场馆管理**：支持多场馆、多场地的统一管理
- **在线预约**：用户可通过小程序选择日期、时段进行场地预约
- **智能推荐**：当期望时段不可用时，系统自动推荐相近时段或替代场馆
- **二维码核销**：用户生成核销码，管理员扫码确认使用
- **违规管理**：自动记录爽约行为，支持违规处罚机制
- **数据统计**：实时展示预约、核销等运营数据

### 项目结构

## 核心代码解析

### 预约服务

**文件位置**: `backend/src/main/java/com/stadium/booking/service/BookingService.java`

![预约服务代码](screenshots/images/code-booking-service.png)

**代码说明**:

预约服务是系统的核心业务逻辑，主要功能包括：

1. **预约验证** (`validateBooking`)
   - 检查用户预约权限
   - 验证时段是否在营业时间内
   - 检查用户当日预约数量限制

2. **冲突检测** (`countConflictingBookings`)
   - 查询同一场地、同一日期、相同时段的预约数量
   - 防止重复预约

3. **预约创建** (`createBooking`)
   - 生成唯一预约单号 (格式: BK + 日期 + 随机码)
   - 计算时段数量
   - 记录审计日志

```java
// 核心逻辑：时段冲突检测
int conflicts = bookingRepository.countConflictingBookings(
    request.getCourtId(),
    request.getBookingDate(),
    request.getStartTime().toString(),
    request.getEndTime().toString()
);

if (conflicts > 0) {
    throw new BusinessException(ErrorCode.SLOT_CONFLICT, "时段已被占用");
}
```

### 核销服务

**文件位置**: `backend/src/main/java/com/stadium/booking/service/CheckinService.java`

![核销服务代码](screenshots/images/code-checkin-service.png)

**代码说明**:

核销服务实现二维码签到功能，主要特点：

1. **二维码Token机制**
   - 用户主动生成核销码
   - Token有效期5分钟
   - 一次性使用，核销后失效

2. **核销窗口验证**
   - 只能在预约开始前N分钟内核销
   - 预约结束后无法核销

3. **核销方式**
   - 扫码核销：用户出示二维码，管理员扫描
   - 手动核销：管理员输入预约单号确认

```java
// 核销窗口验证
LocalDateTime windowStart = bookingStart.minusMinutes(venue.getCheckinWindowBefore());

if (now.isBefore(windowStart)) {
    throw new BusinessException(ErrorCode.CHECKIN_NOT_ALLOWED, 
        "核销窗口未开启，请在开始前" + venue.getCheckinWindowBefore() + "分钟内核销");
}
```

### 智能推荐

**文件位置**: `backend/src/main/java/com/stadium/booking/service/RecommendationService.java`

![智能推荐代码](screenshots/images/code-recommendation-service.png)

**代码说明**:

智能推荐服务在用户期望时段不可用时提供替代方案：

1. **推荐策略**
   - 优先推荐期望时间（评分100分）
   - 其次推荐提前时段（评分 90 - 时段差 × 5）
   - 最后推荐延后时段（评分 85 - 时段差 × 5）

2. **替代场馆**
   - 当指定场馆无可用时段时
   - 自动搜索同类型运动的其他场馆
   - 评分降低20分作为替代方案

```java
// 评分计算逻辑
if (slotDiff == 0) {
    item.setLabel("期望时间");
    item.setScore(100);
} else if (start.isBefore(preferredStart)) {
    item.setLabel("提前" + slotDiff + "时段");
    item.setScore(90 - slotDiff * 5);
} else {
    item.setLabel("延后" + slotDiff + "时段");
    item.setScore(85 - slotDiff * 5);
}
```

### 前端组件

**文件位置**: `miniapp/src/pages/booking/booking.vue`

![前端组件代码](screenshots/images/code-frontend-booking.png)

**代码说明**:

小程序预约页面采用 Vue3 Composition API 开发：

1. **组合式函数设计**
   - `useVenue`: 场馆数据管理
   - `useTimeSlots`: 时段数据加载
   - `useBooking`: 预约提交逻辑

2. **时段选择交互**
   - 点击选中/取消时段
   - 自动填充中间时段（连续选择）
   - 实时显示已选时间范围

3. **响应式状态管理**
   - `ref` 管理基础类型状态
   - `computed` 计算派生状态

```typescript
// 时段选择逻辑
const toggleSlot = (slot: TimeSlot) => {
  if (slot.status !== 'free') return
  
  const index = selectedSlots.value.findIndex(
    s => s.startTime === slot.startTime
  )
  
  if (index > -1) {
    selectedSlots.value.splice(index, 1)
  } else {
    selectedSlots.value.push(slot)
    // 自动填充中间时段
    fillIntermediateSlots()
  }
}
```
